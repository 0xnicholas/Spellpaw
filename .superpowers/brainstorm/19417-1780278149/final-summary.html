核心数据流

  1. 用户在中栏空状态输入需求 → taskStore.createTask()
  2. 用户消息发送 → Agent 开始处理（通过 Pandaria SSE）→ 状态 = 进行中
  3. Agent 直接调用 tool（toolRouter）修改项目/生成内容
  4. Agent turn_end → task 自动变为 待审查
  5. 用户点击任务进入对话 → 查看产出 → 对话中 标记完成 或 继续修改

技术要点

状态管理 新增 taskStore（Zustand + Immer + IDB 持久化）
SSE 每个 task 独立 Pandaria session。新增 useTaskSSE hook 管理 per-task SSE 连接
toolRouter 不变。Agent 在任务中调用 tool 直接修改 projectStore
chatStore 保留不动，但 WorkspacePage 不再直接使用
projectStore 不变。仍管理项目树和选择状态
