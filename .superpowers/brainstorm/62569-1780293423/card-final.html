✅ 画布卡片重新设计 — 最终总结

四种卡片类型 · 独立于树 · Agent + 用户双创建

四种卡片类型

类型 颜色 核心字段 视觉特征
📝 剧本 蓝 title, description, dialogue, duration, location, timeOfDay, status 纯文字卡片，无缩略图，底部元数据条
🎨 美术 黄 title, thumbnail, prompt, tags, styleRef 大缩略图（9:16）+ 提示词 + 标签
📦 产出 紫 title, outputType, summary, sourceTaskId, actions[] 内容摘要 + 操作按钮（批准/拒绝/应用）
👤 人物角色 绿 name, role, age, occupation, personality, appearance, avatar? 圆形头像 + 角色信息卡

架构决策

与树的关系 完全独立，移除 linkedTreeNodeId 和双向同步
创建方式 Agent tool 调用 + 用户画布工具栏/右键菜单
数据模型 CanvasNodeData 改为联合类型（每种 card 不同字段）
渲染 四种独立 Node 组件，按 type 分发渲染
现状迁移 sceneCard → 剧本，assetCard → 移除/合并到美术

边界

  • ❌ 不保留 sceneCard/assetCard 旧类型
  • ❌ 不移除现有画布功能（缩放、拖拽、连线）
  • ❌ 不做卡片间连线逻辑改动
  • ❌ 不做画布上的富文本编辑（双击改标题保留）
  • ❌ 人物角色头像生成功 deferred（字段预留）
确认以上全部后，我写入设计文档并进入实现。
