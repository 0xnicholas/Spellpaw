# 角色妆造师 Skill

## 概述
角色妆造师负责从 `CharacterAssetCard.costumeRequirements[]` 生成正式 `CharacterCostumeCard`。妆造设定包括发型、妆容、服装、配饰、三视图图像版本和用户确认状态，为制作区、表演策略、故事板和视频生成提供统一角色视觉资产。

旧版“从剧本中直接提取妆造细节”的能力仅作为 legacy 辅助信息来源，不再作为主入口，也不得绕过 `CharacterAssetCard` 直接产出下游使用的妆造卡。

## 当前权威输入与输出

**主输入**：

- `CharacterAssetCard`
- `CharacterAssetCard.content.costumeRequirements[]` 中的一项妆造需求
- 可选 `ArtDirectionCard`
- 可选 `WorldviewCard`
- 换装时可选 `baseCostumeCardId`

**唯一正式输出**：

- `CharacterCostumeCard`

## CharacterCostumeCard 正式 Schema

```typescript
interface CharacterCostumeCard extends BaseCard {
  cardId: string;
  cardType: "CharacterCostumeCard";
  type: "character_costume";
  title: string;
  upstreamCards: CardRef[];

  characterId: string;
  characterName: string;
  characterAssetCardId: string;

  costumeId: string;
  costumeName: string;
  formName: string;
  episodeRange: string;
  sceneIds: string[];
  baseCostumeCardId?: string;

  worldAlignment: {
    worldviewCardId?: string;
    ethnicity?: string;
    hairColor?: string;
    eyeColor?: string;
    skinTone?: string;
    facialFeatureNotes?: string;
    userConfirmed: boolean;
  };

  costumeDesign: {
    physicalDetails: string;
    hairstyle: string;
    makeup: string;
    clothing: string;
    accessories: string;
    colorPalette: string[];
    materialNotes: string[];
    consistencyLocks: string[];     // 换装时必须保持一致的面部、发型、体型、标志物等
  };

  promptPackage: {
    modelTargets: Array<"gpt-image-2" | "gemini" | "seedance-5" | string>;
    prompt: string;
    negativePrompt?: string;
    aspectRatio: "16:9";
    layout: "character-design-sheet";
  };

  imageVersions: Array<{
    versionId: string;
    image: ImageReference;
    promptSnapshot: string;
    generatedAt: string;
    qualityChecks: {
      characterConsistency: boolean;
      threeViewCompleteness: boolean;
      expressionActionReadable: boolean;
      styleConsistency: boolean;
      notes?: string;
    };
    userConfirmed: boolean;
    stale: boolean;
  }>;

  selectedVersionId?: string;
  approvedImage?: ImageReference;
  userConfirmed: boolean;
  stale: boolean;
  confirmation?: ConfirmationRecord;
}
```

**硬性规则**：

1. 制作区只读取 `CharacterCostumeCard.cardId` / `costumeId` / `characterId` / `approvedImage` / `selectedVersionId` / `userConfirmed` / `stale`。
2. `userConfirmed !== true` 或 `stale === true` 时，不得作为故事板和视频的权威角色图。
3. 换装卡必须填写 `baseCostumeCardId`，并在 `costumeDesign.consistencyLocks` 中列出必须保持一致的元素。
4. `imageVersions[]` 保存所有生成版本，`selectedVersionId` 指向用户确认版本，`approvedImage` 必须引用同一版本的图。

## 核心职责

### 0. 世界观对齐与种族特征确认（NEW - v1.2.0）
在生成角色妆造前，必须先进行世界观对齐：
- **读取世界观设定**：确认故事发生地、文化背景、主要人口种族
- **确认角色种族特征**：根据世界观推断或确认角色的种族、发色、眼睛颜色、肤色
- **种族信息继承**：将种族信息完整记录到角色妆造资产卡片
- **图像生成约束**：确保三视图提示词明确声明种族特征，避免AI模型默认欧美化

**重要性**：AI图像生成模型（GPT-Image-2、Gemini等）默认倾向生成欧美人面孔。在中国背景的故事中，如果不明确指定种族特征，角色可能出现欧美面孔，严重违反世界观设定。

详见：`/docs/角色资产管线-种族地域特征对齐规范.md`

### 1. 妆造细节提取
从剧本中识别和提取：
- **发型设计**：长度、颜色、造型、质感
- **妆容风格**：底妆、眼妆、唇妆、特殊妆效
- **服装设计**：款式、颜色、材质、风格
- **配饰细节**：首饰、帽子、包包、鞋子等
- **整体风格**：时尚度、年代感、职业特征

### 2. 场景妆造变化
- 识别不同场景下的妆造变化
- 记录角色在不同情节阶段的形象转变
- 标注特殊场景的妆造需求（如婚礼、葬礼、派对等）

### 3. 妆造设定卡片生成
生成结构化的妆造设定文档，包含：
- 角色基础信息
- 日常妆造设定
- 场景妆造变化
- 参考图片描述
- 妆造预算建议

## 工作流程

### 阶段0：世界观对齐检查（NEW - v1.2.0）

在生成角色妆造前，必须先进行世界观对齐：

#### Step 1：读取世界观卡片
```
查询路径：创作分区/世界观卡片.md
提取信息：
- 故事发生地（国家/地区/城市）
- 文化背景
- 主要人口种族
```

#### Step 2：读取或推断角色种族信息

**如果角色设定卡片已有种族信息**：
- 读取角色设定卡片中的"种族与外貌特征"字段
- 检查是否与世界观一致
- 如有冲突，向用户确认

**如果角色设定卡片没有种族信息**：
- 根据世界观自动推断种族特征
- 应用种族映射规则（见下表）
- 向用户展示推断结果并确认

**种族映射规则**：

| 故事发生地 | 默认种族 | 发色 | 眼睛颜色 | 肤色 |
|-----------|---------|------|---------|------|
| 中国大陆 | 东亚人（汉族） | 黑色 | 黑色/深棕色 | 黄种人肤色 |
| 日本 | 东亚人（日本人） | 黑色 | 黑色/深棕色 | 黄种人肤色（偏白） |
| 韩国 | 东亚人（韩国人） | 黑色 | 黑色/深棕色 | 黄种人肤色（偏白） |
| 美国 | 混合（需单独确认） | 多样 | 多样 | 多样 |
| 欧洲 | 白种人（具体国家） | 多样 | 多样 | 白种人肤色 |
| 中东 | 中东人 | 黑色/深棕色 | 棕色/黑色 | 棕色肤色 |
| 非洲 | 黑人 | 黑色 | 黑色/深棕色 | 黑人肤色 |

#### Step 3：记录种族信息到妆造资产卡片

在角色妆造资产卡片中添加"种族与外貌基础"字段：

```markdown
## 种族与外貌基础

**种族**：[从世界观推断或角色设定继承]
**发色**：[黑色/棕色/金色等]
**眼睛颜色**：[黑色/棕色/蓝色等]
**肤色基调**：[黄种人肤色/白种人肤色/黑人肤色/具体描述]
**面部轮廓**：[东亚人面部特征/欧美人面部特征/具体描述]

**面部特征详细描述**：
[根据种族详细描述面部特征，如：
- 东亚人：单眼皮或内双、扁平鼻梁、颧骨略高、面部轮廓柔和
- 白种人：深邃眼窝、高鼻梁、立体五官
- 黑人：宽鼻、厚唇、深色皮肤]
```

#### Step 4：用户确认

向用户展示推断的种族信息：
```
根据世界观设定（故事发生在[地点]），推断角色[角色名]的种族特征为：
- 种族：[种族]
- 发色：[发色]
- 眼睛颜色：[眼睛颜色]
- 肤色：[肤色]

是否确认？（是/否/修改）
```

**检查清单**：
- [ ] 世界观卡片已读取
- [ ] 角色种族信息已确认
- [ ] 种族信息已记录到妆造资产卡片
- [ ] 种族信息与世界观一致

### 输入（legacy 辅助，不是主入口）
- 剧本文本（完整或片段）：只用于补充已缺失的妆造线索
- 角色列表（如有）：只用于核对角色名称，不生成正式 ID

1. **剧本分析**
   - 识别所有角色

2. **细节提取**
   - 发型细节
   - 妆容细节
   - 服装细节
   - 配饰细节

3. **场景映射**
   - 标注妆造出现的场景
   - 记录妆造变化的时间点
   - 分析妆造与情节的关系

4. **设定卡片生成**
   - 整合所有妆造信息
   - 生成结构化文档
   - 添加实施建议

## 图像生成提示词规范（NEW - v1.3.0）

### 概述

为角色妆造三视图生成图像提示词时，必须遵循统一的模板和规范，确保：
1. 风格一致性：所有角色符合项目美术设定
2. 格式一致性：所有提示词结构统一
3. 质量保证：提示词足够详细和精确

### 核心原则

#### 1. 必须参考上游卡片

生成提示词前，**必须**读取并应用以下上游卡片：

**必读卡片**：
- **美术设定卡片**：获取整体视觉风格、色调方案、材质要求、光影参数
- **角色资产卡片**：获取角色基础信息、气质特征

**参考模板**：
- **已有的成功案例**：如果项目中已有其他角色的三视图提示词，必须使用其作为模板
- 保持模板的结构、技术要求、参考作品
- 只替换角色特定的信息（姓名、外貌、服装、气质等）

#### 2. 完整的提示词模板结构

标准提示词必须包含以下所有部分：

```markdown
【开头说明】
- 明确说明这是专业角色设定表（Character Design Sheet）
- 说明用途：影视制作和表演参考
- 说明整体风格和参考作品

【布局说明】
- 左侧主区域（约40%）：三视图布局和姿态描述
- 中央区域（约10%）：干净的角色头像（肩部以上），正面视角，作为视觉中心
- 右上角区域（约25%）：5个典型表情，明确说明"排列成2行3列"
- 右中区域（约15%）：4个典型动作
- 右下角区域（约10%）：4个细节特写

【角色描述】
- 基本信息：姓名、年龄、性别、种族、身高、体型
- 面部特征：眉毛、眼睛、鼻梁、嘴唇、轮廓、肤色
- 发型：样式、颜色、造型、质感、配饰
- 妆容：底妆、眉毛、眼妆、唇妆、整体
- 服装：上装、下装、鞋子、整体风格
- 配饰：首饰、道具等
- 整体气质：姿态、眼神、手势、情感基调

【风格要求】（关键）
- 参考美术设定：引用具体的色调、材质、光影参数
- 服装色彩：具体配色方案和比例
- 材质质感：具体的材质描述
- 光影：色温、对比度、明暗比例等具体数值
- 整体风格：明确的风格描述（如"现代都市写实主义"）

【技术要求】（必须包含）
- 16:9横版构图
- 纯白色背景（#FFFFFF）
- 完整角色可见，不裁切
- 三视图采用正交视角，无透视变形
- 表情和动作特写清晰可辨
- 细节特写聚焦关键元素
- 无Logo、无水印、无文字（除了标注）
- 4K分辨率（3840x2160）
- 适合影视制作和表演设计参考

【参考作品风格】（必须包含）
- 明确列出具体的参考电影或作品
- 说明风格特点（如"写实、真实、电影质感"）
```

### 工作流程

#### Step 1：读取上游卡片

```
1. 读取美术设定卡片
   - 提取整体视觉风格
   - 提取色调方案（主色调、辅色调、比例）
   - 提取材质要求
   - 提取光影参数（色温、对比度、明暗比例）

2. 读取角色资产卡片
   - 提取角色基础信息
   - 提取视觉化转译（表情、动作、气质）
   - 提取妆造需求
```

#### Step 2：查找并参考成功模板

```
1. 检查项目中是否已有其他角色的三视图提示词
2. 如果存在，读取该提示词作为模板
3. 分析模板的结构和格式
4. 提取可复用的部分：
   - 布局说明
   - 技术要求
   - 参考作品风格
   - 风格要求中的美术设定引用
```

#### Step 3：生成新角色提示词

```
1. 使用模板结构
2. 替换角色特定信息：
   - 姓名、年龄、性别
   - 面部特征、发型
   - 服装、配饰
   - 表情、动作
   - 气质特征
3. 保持模板的：
   - 布局说明格式
   - 技术要求
   - 参考作品
   - 美术设定参数
```

### 质量检查清单

生成提示词后，必须检查：

- [ ] 是否读取了美术设定卡片
- [ ] 是否参考了已有的成功模板
- [ ] 是否包含完整的布局说明（4个区域）
- [ ] 表情区域是否明确说明"排列成2行3列"
- [ ] 是否包含完整的技术要求（包括4K、正交视角等）
- [ ] 是否包含【参考作品风格】部分
- [ ] 是否引用了美术设定的具体参数（色调、光影等）
- [ ] 是否明确了视觉风格（如"现代都市写实主义"）
- [ ] 右下角细节特写是否有针对性

### 常见错误与解决方案

#### 错误1：风格描述模糊

❌ 错误示例：
```
- 整体风格：专业角色设计
- 避免水彩风格
```

✅ 正确示例：
```
- 整体风格：现代都市写实主义
- 电影质感，不要网剧的廉价感
- 真实的人物表现，避免过度CG化
- 细节真实：材质、纹理、使用痕迹都要符合真实逻辑

【参考作品风格】
《釜山行》、《行尸走肉》、《寄生虫》的角色设计风格 - 写实、真实、电影质感。
```

#### 错误2：技术要求不完整

❌ 错误示例：
```
【技术要求】
- 16:9横版构图
- 高分辨率
- 写实摄影风格
```

✅ 正确示例：
```
【技术要求】
- 16:9横版构图
- 纯白色背景（#FFFFFF）
- 完整角色可见，不裁切
- 三视图采用正交视角，无透视变形
- 表情和动作特写清晰可辨
- 细节特写聚焦关键元素
- 无Logo、无水印、无文字（除了布局说明中提到的标注）
- 4K分辨率（3840x2160）
- 适合影视制作和表演设计参考
```

#### 错误3：布局描述不精确

❌ 错误示例：
```
右上角区域：5个典型表情
1. 微笑
2. 凝视
3. 注视
4. 专注
5. 微笑
```

✅ 正确示例：
```
右上角区域（占画面约25%）：
典型表情参考，5个头部特写，排列成2行3列：

第一行：
• 柔和微笑 - 眉眼舒展，嘴角温和上扬，眼神充满关怀
• 坚定凝视 - 眉头舒展，眼神坚定有力，嘴角紧抿
• 温柔注视 - 眉眼弯弯，眼神柔和温暖，嘴角微微上扬

第二行：
• 思考专注 - 眉头微皱，眼神专注，略微咬唇
• 自信微笑 - 嘴角上扬，眼神坚定，充满自信

每个表情下方标注表情名称。
```

#### 错误4：未引用美术设定参数

❌ 错误示例：
```
【风格要求】
- 服装色彩：浅色系
- 光影：柔和
```

✅ 正确示例：
```
【风格要求】

**参考美术设定 - 海底基地色调**：
```

## 角色换装功能 (Character Costume Change)

### 概述
角色换装功能允许在保持角色完全一致的前提下，为同一角色创建不同服装版本的三视图卡片。这对于同一角色在不同场景、不同时期、不同场合的妆造需求非常关键。

### 核心原则

#### 1. 角色一致性保证（CRITICAL）
换装的核心挑战是确保换装前后的角色是**同一个人**。必须保持以下元素完全一致：

**必须保持一致的元素**：
- ✅ **面部特征**：眉毛形状、眼睛形状、鼻梁、嘴唇、面部轮廓、肤色
- ✅ **发型**：长度、颜色、造型、质感（除非剧情需要改变发型）
- ✅ **体型比例**：身高、体型、身材比例
- ✅ **典型表情**：角色的标志性表情必须一致
- ✅ **动作姿态**：动作的身体姿态、眼神状态必须一致
- ✅ **特殊标志**：疤痕、纹身、特殊能力的视觉标志等
- ✅ **整体气质**：角色的气场、性格特质的外在表现

**只更换的元素**：
- ❌ **服装**：款式、颜色、材质、风格
- ❌ **配饰**：首饰、帽子、包包、鞋子、装备
- ❌ **整体氛围**：根据场景调整（舒适/警戒/正式等）
- ❌ **色调和光影**：根据场景环境调整

#### 2. 换装的适用场景

**场景换装**：
- 室内 ↔ 室外（休闲装 ↔ 外出装）
- 日常 ↔ 特殊场合（便装 ↔ 礼服）
- 安全区 ↔ 危险区（休闲装 ↔ 作战装备）

**时期换装**：
- 不同季节（夏装 ↔ 冬装）
- 不同阶段（前期 ↔ 后期）
- 角色成长（学生装 ↔ 职业装）

**功能换装**：
- 专业装备（医生服、警察制服、战斗装备）
- 伪装装扮（卧底装、平民装）
- 特殊环境（潜水服、防护服、太空服）

### 换装工作流程

#### Step 1：确定基准妆造
选择一个已有的角色妆造三视图卡片作为基准版本，该版本应该是角色最具代表性的妆造。

```markdown
基准妆造：角色妆造三视图-[角色名]-[妆造名称].md
```

#### Step 2：分析换装需求
明确换装的目的和场景：
- **换装原因**：为什么需要换装？（场景变化、剧情需要、功能需求）
- **目标场景**：新服装将在哪些场景使用？
- **功能需求**：新服装需要满足什么功能？（防护、伪装、礼仪等）
- **时间范围**：适用于哪些集数或场景？

#### Step 3：设计新服装
基于换装需求设计新服装，参考以下因素：
- **角色性格**：服装风格要符合角色性格
- **美术设定**：参考美术设定卡片中的色彩系统、材质指导
- **场景环境**：服装要适应目标场景的环境（色调、光影、氛围）
- **功能性**：确保服装符合使用场景的实际需求

#### Step 4：创建换装版三视图卡片
基于基准妆造创建新的三视图卡片，重点标注以下内容：

**卡片元数据**：
```markdown
**卡片类型**: character_costume_3view
**角色名称**: [角色名]
**妆造名称**: [新妆造名称]
**剧集范围**: [适用集数]
**生成时间**: [日期]
**状态**: draft
**换装基准**: 基于"[基准妆造卡片名称]"换装
```

**换装说明部分**（必需）：
```markdown
## 换装说明 (Costume Change Notes)

### 换装目的
[说明为什么需要换装，适用于哪些场景]

### 角色一致性保证（关键）
**以下元素必须与基准妆造完全一致**：
- ✅ 面部特征（眉毛、眼睛、鼻梁、嘴唇、轮廓、肤色）
- ✅ 发型（[具体描述]）
- ✅ 体型比例（[具体描述]）
- ✅ 典型表情（[列出表情数量]）
- ✅ 典型动作（[列出动作数量]）
- ✅ 特殊标志（[如有]）
- ✅ 整体气质（[具体描述]）

**只更换以下元素**：
- ❌ 服装（[旧服装] → [新服装]）

- ❌ 整体氛围（[旧氛围] → [新氛围]）
```

#### Step 5：编写换装版提示词
在提示词中强化角色一致性要求：

**提示词结构**：

### 技术实现要点

#### 1. 提示词技术
- **多次重复强调**：在提示词中多次重复需要保持一致的元素
- **明确标注**：在每个描述部分明确标注"必须与基准妆造完全一致"
- **对比说明**：提供换装对比表，清晰展示哪些元素变化，哪些保持一致
- **负面提示增强**：在negativePrompt中添加"different person, inconsistent face"

#### 2. 参考图机制（推荐）
在正式生产环境中，建议使用以下技术增强角色一致性：
- **ControlNet参考**：使用基准妆造的生成图作为参考图
- **面部锁定**：使用面部识别技术锁定角色面部特征
- **多次迭代**：生成多个版本，选择最接近基准妆造的版本

#### 3. 人工审核
- 换装后必须进行人工对比检查
- 使用基准妆造和换装版并排对比
- 重点检查面部特征、发型、表情的一致性

### 常见问题与解决方案

#### Q1：生成的换装版角色看起来像不同的人
**原因**：
- 提示词中一致性要求不够明确
- 没有使用参考图机制
- 生图模型对细节理解偏差

**解决方案**：
- 在提示词中多次重复强调一致性元素
- 使用基准妆造的生成图作为ControlNet参考
- 生成多个版本，选择最接近的
- 使用更详细的面部特征描述

#### Q2：表情和动作不一致
**原因**：
- 提示词中没有明确标注表情和动作需要一致
- 动作描述中混入了服装相关的描述

**解决方案**：
- 在表情和动作部分明确标注"必须与基准妆造完全一致"
- 将动作姿态和服装描述分离
- 提供换装前后的动作对比说明

#### Q3：气质发生变化
**原因**：
- 服装风格差异过大导致气质感觉改变
- 光影和色调变化影响了整体气场

**解决方案**：
- 在换装设计时保留角色核心气质元素
- 光影和色调变化不宜过大
- 在提示词中强调"整体气质保持一致"
