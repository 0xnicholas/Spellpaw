## Audit reproduction

The Python script used for this audit (single-pass, word-boundary safe):

```python
import os, re
from collections import defaultdict

CN_RE = re.compile(r'[\u4e00-\u9fff\u3000-\u303f\uff00-\uffef]')
EXCLUDE = ['locales/', '.test.', 'logger.ts']

for root, dirs, files in os.walk('src'):
    dirs[:] = [d for d in dirs if d not in ('node_modules', 'dist', '.git')]
    for f in files:
        if not (f.endswith('.ts') or f.endswith('.tsx')):
            continue
        path = os.path.join(root, f)
        if any(p in path for p in EXCLUDE):
            continue
        with open(path) as fp:
            for line in fp:
                if CN_RE.search(line):
                    print(path, line.rstrip())
```

The categorization into "user UI" / "llm" / "data" was done by path
heuristics in the same script. Paths matching `components/`, `pages/`,
`shared/components/`, `shared/i18n/`, `apps/portal/` are user UI. Paths
matching `systemPrompt.ts`, `intentRouter.ts`, `proactiveInsights.ts`,
`toolRouter.ts`, `chatStore.ts`, `lib/canvasToolkit`, `lib/builderHandlers`
are llm/tool msg. Paths matching `/data/`, `mockTreeData`, `treeDiff.ts`,
`projectStore.ts`, `canvasCardSchema.ts`, `exportPrint.ts` are data/algorithm.
