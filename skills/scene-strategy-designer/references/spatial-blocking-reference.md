# 空间站位与场景调度


## 场景策略的核心问题

场景策略不只是“场景好看”，而是回答：

- 谁在画面左/中/右？
- 谁在前景/中景/后景？
- 谁能看见谁？
- 谁靠近、后退、穿过、被阻隔？
- 动作路径从哪里到哪里？
- 哪些门、窗、桌、墙、车辆、楼梯会参与戏？

## 站位字段建议

```typescript
characterPositions: Array<{
  characterId: string;
  characterName: string;
  depth: "foreground" | "midground" | "background";
  horizontal: "left" | "center" | "right";
  facing: string;
  posture: string;
  eyelineTarget?: string;
  movementPath?: string;
  blockingPurpose: string;
}>
```

## 结构对应站位

多人对话：

- 先锁定轴线。
- 明确说话者、听者和沉默者的位置。
- 给插入镜头预留可拍物件：碗、杯、手机、文件、门把手。

长特写微表情：

- 背景只保留稳定情绪信息。
- 光源、眼神方向和面部可读性优先。
- 不设计复杂走位。

连续动作：

- 明确起点、目标、障碍、转弯点、出口。
- 标记环境反馈点：墙、桌、车门、水面、地面、柱子。
- 保持运动方向和进入/退出方向。

大场面压缩：

- 只选一个主视觉锚点，不让群体抢主角。
- 人群作为压力和背景运动，不当多个主角。
- 场景要有清晰纵深。

## 场景参考图类型

- 纯场景图：稳定空间结构，最好无人物。
- 站位标注图：在场景效果图上标角色位置。
- 首帧参考：控制开场构图和人物初始状态。
- 尾帧参考：控制下一镜头组或视频续写。
- 关系图：控制两人高度、距离、站位和视线。
