卡片与项目树的关系 & 创建方式

卡片从哪来？和树节点是什么关系？

当前机制回顾

现有 sceneCard 有 linkedTreeNodeId，与树节点双向同步。添加场景到树 → 自动创建画布卡片。删除树节点 → 卡片同步删除。

新机制：三种卡片与树的关系？

A

全部独立于树

三种卡片都是画布上的自由卡片，不绑定树节点。用户拖入画布、Agent 创建、用户手动创建。树和画布完全解耦

B

剧本卡片 ↔ 树节点双向同步

剧本卡片保持与树节点的双向同步（现有的 linkedTreeNodeId 机制）。美术卡片和产出卡片是独立的自由卡片，不绑定树

C

三种都绑定树，但有不同关联方式

剧本 ↔ 场景/镜头节点，美术 ↔ 引用树节点（只读关联），产出 ↔ 不绑定特定节点（全局产出）

卡片怎么产生？

D

从画布工具栏拖入

画布顶部有卡片类型选择器，用户拖拽创建空白卡片

E

Agent 在任务中创建

Agent 调用 tool 自动创建卡片（生成分镜 → 美术卡片，分析完成 → 产出卡片）

F

从树节点自动生成

添加场景/镜头到树 → 自动创建对应的剧本卡片到画布（保持现有双向同步）
