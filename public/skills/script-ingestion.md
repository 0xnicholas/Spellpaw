# Script Ingestion Skill

Parses the user's uploaded full script file into the unique canonical entry card `FullScriptCard`. This Skill is only responsible for "reading, parsing, confirming metadata, preserving the original text, and creating the full script card"; it does not split characters/scenes/props. Decomposition is handled by `script-deconstruct`.

## Input

- User-uploaded script file: `.txt`, `.md`, `.docx`, `.pdf`, `.fountain`
- Optional user-supplied info: work type, aspect ratio, duration, episode count, target platform

## Output

- `FullScriptCard`

## Workflow

1. **Read the file**
   - Save `sourceFile.fileName`, `mimeType`, `sizeBytes`, `uploadedAt`, optional `checksum`
   - Preserve original line breaks, scene headings, character names, line indentations, `△` action descriptions, VO/OS markers as much as possible

2. **Parse the text**
   - Generate `rawText`: the complete original script text — never save only a summary
   - Generate `parseVersion`: format `script-ingestion@YYYY-MM-DD`
   - Record `parserWarnings[]`: garbled characters, header/footer residue, unrecognized scene headings, discontinuous episode numbers, etc.

3. **Infer and confirm metadata**
   - `metadata.projectType`
   - `metadata.aspectRatio`
   - `metadata.duration`
   - `metadata.episodeCount`
   - Optional `metadata.targetPlatform`
   - If the user skips, defaults may be used, but must record "to be confirmed" in `parserWarnings` or `notes`

4. **Create `FullScriptCard`**
   - `id` is a unique card ID
   - `type = "full_script"`
   - `upstreamCards = []`
   - `content.sourceFile`, `content.rawText`, `content.parseVersion` are required
   - `content.episodes` can be a preliminary parsed structure; if the original script cannot be reliably split into episodes, an empty array is allowed, but `rawText` must not be empty

5. **Hand off to script deconstruct**
   - After user confirmation, call `callSkill('script-deconstruct', { fullScriptCard })`
   - The old name `script-deconstruction` is forbidden

## FullScriptCard Minimal Structure

```typescript
interface FullScriptCard extends BaseCard {
  type: "full_script";
  upstreamCards: [];
  content: {
    title: string;
    subtitle?: string;
    author?: string;
    version: string;
    format: "feature" | "series" | "short";
    totalEpisodes?: number;
    estimatedDuration?: string;
    sourceFile: {
      fileId: string;
      fileName: string;
      mimeType: string;
      sizeBytes?: number;
      uploadedAt: string;
      checksum?: string;
    };
    rawText: string;
    parseVersion: string;
    parserWarnings?: Array<{
      code: string;
      message: string;
      location?: string;
    }>;
    metadata: {
      projectType: "电影" | "电视剧" | "短剧" | "广告" | "MV" | "纪录片";
      aspectRatio: "16:9" | "9:16" | "1:1";
      duration: {
        perEpisode?: number;
        total?: number;
      };
      episodeCount: number;
      targetPlatform?: string[];
      videoGenerationProfile?: VideoGenerationProfile;
    };
    episodes: Array<{
      episodeNumber: number;
      episodeTitle?: string;
      scenes: Array<{
        sceneNumber: string;
        heading?: string;
        rawTextRange?: {
          startOffset: number;
          endOffset: number;
        };
      }>;
    }>;
    characters?: Array<{ name: string; description?: string }>;
    locations?: Array<{ name: string; description?: string }>;
    productionNotes?: string;
  };
}
```

## Hard Rules

- Do not create a `FullScriptCard` if `rawText` is empty.
- Do not create a `FullScriptCard` if `parseVersion` is empty.
- On parse failure, return errors and `parserWarnings` first; do not let downstream Skills guess.
- `script-ingestion` only creates the entry card; all creation-zone downstream cards are generated by `script-deconstruct`.

## Next Step After Completion

Completion criteria: `FullScriptCard` has been created with `sourceFile`, `rawText`, `parseVersion`, and user-confirmed metadata complete.

Prefer calling `script-deconstruct` to decompose the full script into episode-scene tables, character/scene/prop settings, art direction, and music direction cards.

Suggested message: `The full script card has been created. I recommend calling script-deconstruct next to begin script decomposition. Shall we continue?`
