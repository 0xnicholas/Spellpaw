任务内的 Agent 行为 & 数据结构

Agent 在任务对话中做什么？任务存储哪些信息？

先确认：任务的数据结构

每个任务至少需要：

id 唯一标识
title 任务标题（Agent 根据对话自动生成）
status in_progress | pending_review | completed
messages 对话消息列表（和当前 ChatMessage 类似）
createdAt 创建时间

还需要其他字段吗？比如关联的画布节点、输出产物等？

关键行为问题：Agent 在任务中是直接执行还是先提议？

A

Agent 直接执行，结果进入「待审查」

Agent 在对话中直接调用 tool（添加场景、生成分镜等），执行完毕后任务自动变为「待审查」状态。用户审查后可批准或回退

B

Agent 先提议，用户确认后执行

Agent 先输出建议方案（不修改项目），用户确认「执行」后 Agent 再调用 tool 实施。类似 Plan → Execute 模式

C

取决于任务类型

生成类任务（分镜图）直接执行；修改类任务（调整结构、改对白）先提议再确认
